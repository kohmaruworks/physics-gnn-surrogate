# サロゲートモデル構築(2)：スケール変動テスト — 構造を持たないAI（MLP）のクラッシュ証明

## 1. はじめに

研究用のノートブックでは、ノード数やメッシュが固定された「都合の良いデータセット」で学習させやすい。しかし実務の R&D では、**メッシュの切り直し、局所リファインメント、トポロジ変更**により、離散化後の自由度の数 $N$ は固定ではない。サロゲートを現場に載せるには、**訓練時と推論時で $N$ が変わっても破綻しないか**が実用上の壁になる。

本稿では、意図的に極端な設定を取り、**訓練想定 $N=10$ で構築した MLP**に、**$N'=50$ の未知システム**を入力したときに何が起きるかを整理する。デモ用スクリプト `demo1_scale_generalization.py` と同型の論理である。

## 2. 実務の壁：スケールの変動（Zero-Shot Generalization）

「Zero-Shot」とは厳密には学習分布外の多様な一般化を指すが、ここでは次の**サイズ一般化**に焦点を当てる。

- モデル設計時：チェーングラフのノード数 $N=10$ に合わせ、MLP の第1層は入力ベクトル長 $Nd$（特徴次元 $d$）に固定する。
- 推論時：同じ物理（チェーン）だがノード数 $N'=50$ のデータが与えられる。

GNN は `edge_index` の列数が増えるだけで、**層のパラメータ形状は $N$ に依存しない**（ノードごとに同じ変換を共有する）。一方、上記 MLP は**入力ベクトル長が設計時に固定**されている。

```python
# プレースホルダ：N=10 専用 MLP の第1層
# nn.Linear(10 * d, hidden_dim)
```

```python
# プレースホルダ：N=50 の特徴をそのまま flatten すると長さ 50*d
# x_flat = x.view(1, -1)   # shape (1, 50*d)
# model(x_flat)  # -> RuntimeError（下節）
```

## 3. 検証結果と数学的考察：MLPのクラッシュ

### 3.1 行列積の次元条件

第1層が全結合で、バッチを行ベクトル $\mathbf{x}^{\mathsf{T}} \in \mathbb{R}^{1 \times (N'd)}$ とおく。重み行列を $W^{(1)} \in \mathbb{R}^{(Nd) \times h}$（設計時 $N$、隠れ次元 $h$）とすると、順伝播の線形部分は

$$
\mathbf{z} = \mathbf{x}^{\mathsf{T}} W^{(1)}
$$

である。行列積が定義されるためには**内側の次元が一致**する必要があり、

$$
N'd = Nd
$$

でなければならない。$d$ が正なら $N' = N$ が必要である。**$N' \neq N$ のとき積は定義されず**、PyTorch では `RuntimeError: size mismatch` として表れる。

実際のエラーメッセージ例は、$(1 \times N'd)$ と $(Nd \times h)$ を掛けようとして失敗する形である。

```
# 例（N=10, d=2, N'=50 のとき）:
# mat1 and mat2 shapes cannot be multiplied (1x100 and 20x32)
```

ここで $N'd = 100$, $Nd = 20$ である。

### 3.2 GNN がスケールフリーに推論できる理由（概要）

GCN などのメッセージパッシングは、各層で**ノードごとに同じ変換**を適用し、**辺に沿って近傍から集約**する。パラメータは「チャネル間の線形変換」と「グラフに依存しない正規化」に載るため、**ノード数が増えてもテンソル形状 $(N, d_{\mathrm{out}})$ は合法**に保たれる（`edge_index` がそのグラフに対応していればよい）。

$$
X^{(\ell+1)} = \sigma\!\left( \mathrm{AGG}\bigl( A,\, X^{(\ell)} W^{(\ell)} \bigr) \right)
$$

のように書ける場合、$W^{(\ell)}$ の形状は $N$ に依存せず、隣接構造 $A$（稀疏表現）が $N'$ に合わせて与えられれば前向きは実行できる。

### 3.3 デモ実行（プレースホルダ）

```bash
# .venv/bin/python demo1_scale_generalization.py
```

成功時、GNN は `[Success]` と形状を表示し、MLP は `try-except` で捕捉した `RuntimeError` を表示する想定である。
