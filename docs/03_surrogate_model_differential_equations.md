# 第3章：微分方程式のGNNサロゲートモデリング

本章では、Julia の DifferentialEquations.jl によって生成した **物理に整合した時間発展** を教師信号とし、PyTorch Geometric 上のグラフニューラルネットワーク（GNN）がそれを回帰的に近似するパイプラインを説明する。いわゆる **サロゲート（代理）モデル** の最小構成例として、1 次元バネ–質量チェーンの **初期状態から短時間後の状態への写像** を学習対象とする。

対象コードは `spring_mass_chain_export.jl`（ODE 求解と JSON 出力）、`export_catlab_graph_json.jl`（`x` / `y` のシリアライズ）、`import_catlab_json_to_pyg.py`（`Data` への復元）、および `train_spring_mass_gcn.py`（教師付き学習）である。

---

## 1. 物理法則の厳密解生成（Julia / DifferentialEquations.jl）

### 1.1 2 階 ODE から連立 1 階系への還元

1 次元直列バネ–質量系において、各質点 \(i\) の変位を \(u_i(t)\)、速度を \(v_i(t) = \dot{u}_i(t)\) とする。隣接質点間のバネ定数を \(k\)、質量を \(m\) とし、端は自由（壁なし）とする。フックの法則に基づき、加速度は次のように書ける。

- 端の質点: \( \ddot{u}_1 = \frac{k}{m}(u_2 - u_1) \)、\( \ddot{u}_n = \frac{k}{m}(u_{n-1} - u_n) \)
- 内部: \( \ddot{u}_i = \frac{k}{m}(u_{i+1} - 2u_i + u_{i-1}) \)

これは **\(n\) 自由度の連立 2 階常微分方程式系**である。一般に SciML 系のソルバは **1 階の \(\dot{z} = f(z,t)\)** 形を前提とするため、状態ベクトルを

\[
z(t) = \bigl(u_1,\ldots,u_n,\, v_1,\ldots,v_n\bigr)^{\mathsf{T}} \in \mathbb{R}^{2n}
\]

と定義し、

\[
\dot{u}_i = v_i,\qquad \dot{v}_i = \text{（上記加速度式）}
\]

とおく。`spring_mass_chain_export.jl` の `spring_mass_chain_ode!` は、この **連立 1 階系の右辺**を in-place で `dz` に書き込む実装である。パラメータはタプル `p = (n, k, m)` として `ODEProblem` に渡され、質点数と物理定数を_rhs に閉じ込めている。

初期条件は、ノード特徴行列 `x0` の第 1 列を位置、第 2 列を速度とみなし、

\[
z(0) = \bigl(x_{1,1},\ldots,x_{n,1},\, x_{1,2},\ldots,x_{n,2}\bigr)^{\mathsf{T}}
\]

として `z0 = vcat(x0[:, 1], x0[:, 2])` により構築する。

### 1.2 `ODEProblem` と `Tsit5` による短時間積分

`integrate_spring_mass_chain` は `ODEProblem(spring_mass_chain_ode!, z0, (0.0, t1), p)` を定義し、`solve` に **Tsitouras 5 次ルンゲ–クッタ系ソルバ `Tsit5`** を指定する。`saveat = t1` により、積分区間 \([0, t_1]\) の終端（既定では \(t_1 = 0.1\)）における状態を取得する。許容誤差 `abstol` / `reltol` を小さく設定することで、**参照用の高精度な数値解**を得る意図である。

ここで言う「厳密」とは、閉形式の解析解を指すのではなく、**与えられた ODE とパラメータに対し、適応的な高次ルンゲ–クッタ法と誤差制御によって得られる、物理法則に忠実な数値軌道**を意味する。これを下流の機械学習が **教師（ラベル）** として用いる。

### 1.3 サロゲートモデリングの考え方

数値積分はステップごとに右辺評価と代数演算を要し、大規模系・長時間・パラメータスイープではコストが問題になることがある。**サロゲートモデル**は、高忠実度シミュレータの入出力関係を、より安価な近似器（ここでは GNN）で置き換えるアプローチである。

本実装では、写像

\[
\Phi:\ x \mapsto y
\]

を、ODE ソルバによって **真の \(\Phi\)**（\(t=0\) の \((u,v)\) から \(t=t_1\) の \((u,v)\) へ）として離散的に計算し、その出力を `y` として保存する。GNN は **同じグラフ上の `x` を入力とし、`y` に近いベクトル場を出力する**ように学習する。完全な時間積分器の置換ではなく、**単一の時間ステップ（または短ホライズン）の効果**を学習する最小例として位置づけられる。これを積み重ねることで、より長いホライズンへの拡張や、異なる \(k,m\) への汎化といった発展課題につながる。

## 2. 圏論的データ構造と物理シミュレーションの統合

### 2.1 単一 JSON にまとめる設計意図

機械学習パイプラインでは、**入力**（グラフ構造とノード特徴）と**教師**（各ノードの目標ベクトル）を一貫した形式で配布することが望ましい。`export_catlab_graph_json.jl` の拡張により、`catlab_graph_to_dict` / `save_catlab_graph_json` はオプション引数 **`y`** を受け取り、JSON オブジェクト内に **`x` と並列のキー**として書き出す。

同一ファイル内に以下を保持する。

- **`format` / `num_nodes` / `edges`**: Catlab で構築した **離散的な結合構造**（PyG の `edge_index` と整合する 0-based の辺リスト）。
- **`x`**: シミュレーションの **初期条件**（各ノードの位置・速度）。
- **`y`**: ODE を \([0,t_1]\) で積分した結果の **終端状態**（同じく位置・速度の 2 次元）。

スキーマ識別子 `catlab_directed_graph_v1` は変更せず、**後方互換**を保ちつつオプションキー `y` で拡張する。これにより、教師なしのグラフのみの JSON と、教師付きデータセットを **同じローダ**（`import_catlab_json_to_pyg.py`）で扱える。`y` が無い場合は `train_spring_mass_gcn.py` がダミー教師にフォールバックする実装となっており、開発段階の段階的移行にも対応している。

### 2.2 圏論的グラフとシミュレーションの役割分担

Catlab の `Graph` は、バネ結合の **トポロジー**（どの質点が相互作用するか）を宣言的に表す。一方、**連続時間の力学**はグラフの型だけでは決まらず、`spring_mass_chain_ode!` が **連続状態 \(z(t)\)** の上で定義される。JSON への統合は、

- **離散構造**（頂点・有向辺）→ GNN のメッセージパッシングの骨格
- **連続ダイナミクス**のサンプル（\(t=0\) と \(t=t_1\) のペア）→ 教師付き学習のペア \((x,y)\)

という二層を **一つの学習サンプル**としてパッケージ化する。圏論的データ構造は「何が結ばれているか」を固定し、微分方程式は「結ばれた系が時間とともにどう動くか」を与える。両者を同一アーティファクトに載せることで、再現性とデータリネージ（出所の追跡）が明確になる。

## 3. GNNによる物理ダイナミクスの自己学習（Python / PyTorch Geometric）

### 3.1 `data.y` をターゲットとした学習ループ

`import_catlab_json_to_pyg.py` は JSON に `y` が存在するとき、`data.y` に `float32` テンソルとして読み込む。`train_spring_mass_gcn.py` では、`getattr(data, "y", None)` が非 `None` のとき **`target = data.y`** とし、Julia が積分した **真の物理ダイナミクスに対応するノードラベル**を損失に用いる。

学習ループの骨格は次のとおりである。

1. `data` を `device`（CUDA 利用可能なら GPU）へ転送する。
2. `TwoLayerGCN` により `out = model(data.x, data.edge_index)` を計算する。各 `GCNConv` は隣接ノードからの集約と線形変換を行い、`ReLU` により非線形性を挿入する。
3. `MSELoss(out, target)` でノードごとの 2 次元ベクトルの平均二乗誤差をスカラー化する。
4. 逆伝播と `Adam` によるパラメータ更新を 100 エポック繰り返す。

入力 `data.x` は固定されたままパラメータのみが更新される **教師あり回帰**である。モデルは「与えられたグラフと初期 \((u,v)\) に対し、短時間後の \((u,v)\) を予測する」写像を、データ駆動で調整する。

### 3.2 数値積分とメッセージパッシングの対応と工学的価値

厳密な意味で **ルンゲ–クッタ法と GCN の層が同型**であるわけではない。ルンゲ–クッタは時間方向に離散化された積分則であり、右辺 \(f(z,t)\) の評価回数とステップ幅が誤差を決める。一方、GCN は **空間方向**に、グラフの辺に沿って隣接ノードの特徴を重み付き集約し、チャネル間の線形結合と活性化で変換する。

工学的には、次の価値が期待される。

- **高速推論**: 一度学習すれば、前向きは行列演算とスパースな近傍集約に帰着し、細かい適応ステップを要しない場合がある（ただし精度・外挿性は課題である）。
- **微分可能性**: PyTorch 経由で勾配が取得できるため、設計変数や制御入力に対する感度解析、あるいはより大きな最適化ループへの組み込みがしやすい。
- **離散トポロジとの自然な結合**: 結合構造が変化しても、同じアーキテクチャで **可変サイズのグラフ**に対する前向きが定義しやすい。

本書の例は **ごく小さな系と浅い GCN** に留まるが、**高忠真度シミュレータが生成した \((x,y)\) を教師とし、GNN がその写像を近似する**という枠組みは、より大規模なマルチボディ系や PDE 離散化系へのサロゲート構築の原型となる。数値積分アルゴリズムをそのまま置き換えるのではなく、**物理と整合したデータから学習した空間的・非線形な近似器**として位置づけることが、実務上の健全な期待値である。

## 4. 実行上の依存関係（整理）

- Julia 側: `Catlab`、`DifferentialEquations`、`JSON3`（および `Project.toml` / `Manifest.toml` に記載の推移的依存）。
- 手順: `julia --project=. spring_mass_chain_export.jl` で `spring_mass_chain_5.json` を生成したうえで、`python train_spring_mass_gcn.py` を実行する。

以上が、微分方程式による教師生成と GNN によるサロゲート学習を一つのワークフローに統合した実装の技術的要約である。
