# Physics-Informed HeteroGNN Surrogate

## Overview

This repository demonstrates a neural surrogate pipeline for multiphysics computational fluid dynamics (CFD): **physics-constrained heterogeneous graph neural networks (HeteroGNN)** built with **Discrete Exterior Calculus (DEC)**-compatible topology exports and **PyTorch Geometric**.

Instead of collapsing the simulation mesh into a single homogeneous adjacency structure, we represent **Primal and Dual complexes** explicitly—respecting where discrete fields naturally live—and train a **heterogeneous message-passing architecture** (**`HeteroConv`**) to propagate information across primal vertices, dual cells, and primal–dual incidence relations.

The scientific goal is to reduce the marginal cost of forward simulation while preserving topological priors relevant to continuum limits; the engineering goal is a **clean language boundary**, **deterministic interchange schema**, and **regression-tested index semantics** suitable for reproducible OSS workflows.

---

## Key Features

- **Strict language boundary (Julia / Python)**  
  **Julia** serves as the source of truth for mesh generation and physics-motivated discrete dynamics; **Python (PyTorch Geometric)** handles representation learning, training, inference, and visualization. The two stacks communicate through a single, versioned **`physics_gnn_interim_v2`** JSON **contract**, not shared in-process coupling.

- **Index safety across the FFI-style boundary**  
  Julia indexing is **1-based**; NumPy / PyTorch are **0-based**. All conversions (including consistent COO conventions for **`edge_index`**) are applied **exactly once** at export time in **`utils_export.jl`**, minimizing the class of silent topology misalignment bugs that otherwise dominate heterogeneous mesh pipelines.

- **Modularity and compositionality-oriented design**  
  The codebase is deliberately factored—mesh/discretization utilities, exporters, loaders, trainers, rollout, and plotting—consistent with **compositionality** idioms emphasized in applied category theory workflows (e.g., future integrations with categorical DSLs for multiphysics). Components can be swapped or extended without rewriting unrelated layers.

- **Topology-first heterogeneous graphs**  
  Edge types approximate **within-Primal adjacency**, **within-Dual adjacency**, and **Primal→Dual incidence** (`primal_to_dual`), enabling bidirectional heterogeneous message passing rather than lumped homogeneous approximations.

- **Efficient heterogeneous convolutions**  
  The model **`PhysicsHeteroGNN`** composes **`HeteroConv`** with **`SAGEConv`** blocks. Only the forward primal→dual **`edge_index`** is stored on disk; the dual→primal tensors are regenerated by transposition inside the forward pass to reduce redundancy and maintain consistency.

- **Physics-informed surrogate loss (current implementation)**  
  A lightweight regularizer loosely encourages near–**divergence-free** kinematics for incompressible regimes: primal velocity channels are **`scatter_add`–aggregated to dual cells** so that approximate **cell-wise flux balance** aligns with \(\nabla \cdot \mathbf{v} = 0\) *in spirit*—not a DEC-faithful discrete Hodge/star operator formulation. Treat this term as an **engineering placeholder** amenable to stricter discrete operators in future revisions.

- **Closed-loop evaluation**  
  Beyond one-step supervised error, inference supports **autoregressive rollout**, surfacing accumulation of model error under feedback—where short-horizon optimum does not imply long-horizon stability.

## Quick Start

All commands assume the **repository root** as the current working directory.

### 1. Generate reference data (Julia)

Julia ≥ 1.9. For toolchain setup and `Pkg.instantiate()`, consult **[docs/julia_setup.md](docs/julia_setup.md)**.

```bash
julia --project=src/julia -e 'using Pkg; Pkg.instantiate()'
julia --project=src/julia src/julia/03_simulation.jl
```

This writes **`data/interim/v2_step1_ground_truth_toy.json`** with schema **`physics_gnn_interim_v2`**.

### 2. Train the surrogate (Python)

Python ≥ 3.10 recommended.

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python src/python/train.py
```

Checkpoints default to **`data/interim/hetero_gnn_model.pth`**.

Example overrides:

```bash
python src/python/train.py --epochs 50 --history-len 1 --lambda-phys 0.1
```

With [`uv`](https://docs.astral.sh/uv/):

```bash
uv venv .venv && source .venv/bin/activate && uv pip install -r requirements.txt
```

### 3. Autoregressive inference and visualization (Python)

```bash
python src/python/inference.py
python src/python/visualize.py --animate
```

Typical artefacts: rollout tensors under **`data/interim/`**, comparison figure **`comparison_plot.png`**, and **`rollout_animation.gif`** when **`--animate`** is set. The inference script prints wall-clock timings (order of **milliseconds per rollout step**) for surrogate-vs-solver amortization narratives.

(Optional) Activate both Python and Julia from a single bash session:

```bash
source scripts/setup_env.sh
```

### Containerized Julia workflow

```bash
docker build -t hetero-surrogate-julia .
docker run --rm -v "$PWD":/app -w /app hetero-surrogate-julia
```

See **`Dockerfile`** for the exact entrypoint (`03_simulation.jl` by default).

## Results and Insights

- **Latency.** On commodity GPUs, amortized surrogate forward time can reach millisecond-scale **per rollout step**, offering orders-of-magnitude headroom versus full nonlinear solves when the bottleneck is repeated forward querying.

- **Autoregressive error growth.** Teacher-forcing-style optimization can yield extremely low single-step mismatch; however, feeding predictions back into the integrator (**autoregressive rollout**) exposes accumulating drift—a canonical pathology for learned dynamical surrogates. Empirically on the bundled toy meshes, horizons on the order of **ten autoregressive steps** already reveal subtle anomalies (e.g., vector-field orientation inconsistencies, accelerated scalar diffusion akin to numerical viscosity in the surrogate channel). These observations motivate multi-step objectives, stabilized training curricula, uncertainty-aware truncation, or structurally richer physics penalties beyond the present **`scatter_add` flux proxy**.

- **Schema discipline.** Robustness hinges on enforcing the interim contract: renaming or hand-editing **`schema`** strings without aligning loader expectations will abort or silently degrade heterogeneous edges—by design rather than ambiguity.

Future work directions include unstructured industrial meshes, fidelity upgrades to DEC-consistent divergence operators inside the loss, 3-D extensions, and CI snapshots spanning Julia export and Python parity tests.
